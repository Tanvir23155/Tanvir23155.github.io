Reverse Engineering Project

Engine-Level Resource Modification — Native C++ Layer

Author Tanvir Date May 2024 Target Library libcocos2dcpp.so Targets ARM64-v8a · ARMEABI-v7a

📋 Executive Summary § 1

This project details the successful reverse engineering and modification of the Mini Militia Classic C++ engine (libcocos2dcpp.so). By targeting the native layer rather than the Smali/Java layer, we achieved stable, engine-level patches for:

🔫 Unlimited Ammunition
Ammo count frozen at max — synced across LAN/Multiplayer
⚡ Unlimited Nitro
Jetpack fuel frozen at 100% — STR instruction replaced with NOP
🔄 Infinite Reserve
Spare ammo count never decrements on reload
📱 Dual Architecture
Supports 64-bit (ARM64-v8a) and 32-bit (ARMEABI-v7a)
ℹ️

The project specifically addresses challenges in LAN/Multiplayer synchronization and multi-architecture compatibility (supporting both modern 64-bit and legacy 32-bit devices).

🔎 Phase I — Symbol Discovery & Analysis § 3

The core gameplay logic is encapsulated in a shared object library. We utilized symbol extraction to identify "mangled" C++ function names responsible for resource management.

Identify Ammunition Subtraction Logic

Shell Script
$ nm -D libcocos2dcpp.so | grep "subAmmo"

Identify Nitro (Power) Management Logic

Shell Script
$ nm -D libcocos2dcpp.so | grep "setPower"

Identified Target Functions

# Function Purpose
1 Weapon::subAmmo(int) Handles bullet reduction (called every shot)
2 SoldierLocalController::setPower(float) Manages fuel levels for Nitro/Jetpack
3 Weapon::completeReloadWeapon() Manages clip transitions on reload

🩹 Phase II — Native Binary Patching (Ghidra) § 4

4.1 — Unlimited Ammunition (LAN/Multiplayer Sync)

PropertyValue
LocationWeapon::subAmmo
Offset0x00a482e0
LogicMaster Subtractor — called during both Solo and LAN play
🩹

Patch: Replace the function entry instruction with RET

  • 64-bit patch: C0 03 5F D6
  • 32-bit patch: 1E FF 2F E1

Result: The function returns immediately before subtraction occurs, ensuring ammo counts remain synchronized at maximum across the network.

4.2 — Pro-Style Reloading (Infinite Reserve)

PropertyValue
LocationWeapon::completeReloadWeapon
🩹

Patch: Identify the call to setAmmo(this, count, false) at the end of the function. Replace the BL (Branch Link) instruction with NOP.

Result: The gun reloads and empties its clip normally, but the "Spare Ammo" count never updates/decreases.

4.3 — Unlimited Nitro (Jetpack)

PropertyValue
LocationSoldierLocalController::setPower
Offset0x009e68c4
🩹

Patch: Identify the STR (Store) instruction that writes the fuel value to memory. Replace with NOP.

Result: The update to the fuel variable is skipped; Nitro remains frozen at 100%.

📱 Phase III — Multi-Architecture Compatibility § 5

To ensure the mod works for friends with older devices, the patches must be applied to both libraries contained within the APK.

64-bit (arm64-v8a)

Uses RET for returns.
Target for modern devices (e.g., Nothing Phone).

32-bit (armeabi-v7a)

Uses BX LR for returns.
Target for legacy/budget devices.

⚠️

Best Practice: If distributing to a group, mod both files and package them into a "Universal" APK. If only 64-bit is modded, 32-bit users will experience the original, un-modded gameplay.

📦 Phase IV — APK Reconstruction & Signing § 6

After exporting the patched .so files from Ghidra and replacing them in the lib/ directory:

6.1 — Building the APK

AAPT2 is required for modern Android resource compatibility.

Shell Script — Build APK
$ apktool b --use-aapt2 mod_project -o MiniMilitia_Mod.apk

6.2 — Signing & Verification

Android will reject unsigned packages. We utilize a private .jks keystore.

Shell Script — Sign the Package
$ apksigner sign --ks my-release-key.jks --out final-modded.apk MiniMilitia_Mod.apk
Shell Script — Verify Signature Integrity
$ apksigner verify final-modded.apk
🔑

Signatures must be unique for the user's keystore. Always keep your .jks file secure and backed up.

🔧 Troubleshooting & Maintenance § 7

  • ⚠️ YAML Metadata Parse Failure
    If apktool.yml fails to parse, remove the top line (often a Windows/Linux path conflict).
  • ⚠️ LAN Desync — Ammo Decreases Only for Host
    Ensure the Host is running the modded APK and that the subAmmo patch is correctly applied with a RET.
  • ⚠️ Installation Failure
    Always verify the original game is uninstalled before installing the modded APK. Signatures must be unique for the user's keystore.
